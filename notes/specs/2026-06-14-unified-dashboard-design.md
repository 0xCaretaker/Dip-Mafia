# Unified Dip Mafia Dashboard - Design Spec

**Date:** 2026-06-14
**Status:** Approved (brainstorming) → ready for implementation
**Branch:** `unified-dashboard`

## Problem

Today there are two disconnected web surfaces with different design languages:

1. **six7 Almanac** - `docs/index.html`, fed by `docs/data.js` (generated by `analysis/build_web.py`). Polished editorial dark theme. Already has a `Full/10y/5y/3y/1y` horizon toggle on its leaderboard.
2. **Strategy Dashboard** - `docs/strat.html`, generated wholesale as HTML by `analysis/portfolio_view.py`. Generic dark/purple Chart.js theme. Three tabs (Portfolio · Backtest · Iterations). Mostly all-time only; horizon data is buried in tables at the bottom of two tabs.

They overlap conceptually ("Timed HODL vs SIP vs NIFTY over horizons") but feel like unrelated products. Two complaints:
- **Structure/looks:** the strategy dashboard is messy and visually generic; the two pages don't cohere.
- **No horizon toggling:** Portfolio and Backtest show all-time graphs/numbers only, not 1Y/3Y/5Y views.

## Decisions (locked)

- **One unified site (Option A).** A single page with one design system, one nav, and one global horizon control driving every section.
- **Portfolio gets a real reconstructed NAV curve (Option 3)** - not a buy-date filter. Compute shares-held-over-time × historical daily close to get an actual portfolio value series, then derive true horizon returns + an equity curve for the Portfolio section.
- **Global horizon set: `1Y / 3Y / 5Y / 10Y / All`.** 10Y parity added across all sections (six7 already has it; strat backtest is 16y so 10Y is computable).
- **Python becomes a pure data layer.** `portfolio_view.py` and `build_web.py` stop emitting HTML and emit data files only. The unified HTML/CSS/JS is hand-authored and **designed by /impeccable** - full latitude on the visual language (no constraint to preserve the current Almanac look).
- **Chart rendering:** /impeccable's call during the build - port to a hand-built SVG chart kit (best cohesion) vs. a themed Chart.js (faster). Lean SVG.

## Information Architecture

Single page `docs/index.html`. Sticky header: brand + section nav + global `1Y/3Y/5Y/10Y/All` segmented control. Five sections, all horizon-aware:

1. **Overview** *(new)* - live portfolio value & P&L + strategy verdict at the chosen horizon. The headline "so what."
2. **Portfolio** - reconstructed NAV equity curve (horizon-sliced), re-scoped summary cards, allocation donut, holdings table with per-stock since-buy return.
3. **Backtest** - Timed HODL vs SIP vs NIFTY equity + drawdown sliced to the window; metric cards & comparison table from `strategy_horizons[H]`; regime / cash / rolling-alpha / buy-distribution / monthly-deployment charts.
4. **Screens (six7)** - leaderboard + inspect-a-list, per horizon (folds in today's Almanac).
5. **Iterations** - watchlist-vs-watchlist comparison table, overlaid equity, horizon return tables.

A section that genuinely lacks a horizon disables that pill for that section (target: none, once 10Y parity lands).

## Data Contract

The frontend reads two globals from two script files. Generators own these; the frontend never reads CSV/JSON directly.

### `docs/data.js` → `window.SIX7_DATA` (unchanged shape; `build_web.py`)
Existing six7 almanac payload: `snapshot_date, generated_at, universe, caveat, horizons[], list_order[], meta{}, comparison{hz:[rows]}, curves{}, ...`. Reused as-is for the **Screens** section. Verify it carries the `10y` horizon (it does today).

### `docs/strat_data.js` → `window.STRAT_DATA` (new; `portfolio_view.py`)
```
{
  generated_at, current_run, end_date,
  portfolio: {
    summary: { total_invested, total_value, total_pnl, total_ret, winners, losers,
               best_stock, best_ret, worst_stock, worst_ret, count },
    rows: [ { stock, avg_price, cmp, shares, invested, value, pnl, ret, weight,
              num_buys, first_buy, last_buy, trades:[{date,price,amount}] } ],
    alloc: { labels:[], values:[] },
    nav: {                                  # NEW - reconstructed portfolio value series
      dates: [], value: [], invested: [],   # daily; invested = cumulative cost basis to date
    },
    horizon: {                              # NEW - per-horizon scoped portfolio metrics
      "1Y": { ret, value, invested, pnl, xirr }, "3Y": {...}, "5Y": {...},
      "10Y": {...}, "All": {...}
    }
  },
  backtest: {
    assumptions: {...}, total_invested,
    summary: { n_signals, n_stocks_bought, ... },
    strategy_colors: {},
    equity:   { <series>: {dates:[], values:[]} },   # full daily; JS slices to horizon
    drawdowns:{ <series>: {dates:[], values:[]} },
    cash_util:{...}, regimes:[...], regime_strategies:[...],
    rolling_alpha:{...}, buy_distribution:[...], buy_timeline:[...],
    horizon_metrics: {                      # EXPANDED strategy_horizons - per horizon × strategy
      horizons: ["1Y","3Y","5Y","10Y","All"],
      strategies: ["Timed HODL","SIP","NIFTY 50"],
      cells: { "<strategy>|<H>": { final_value, mult, xirr, sharpe, sortino,
                                   maxdd, maxdd_days, vol, cash } }
    }
  },
  iterations: [ { label, is_current, watchlist, n_stocks, period,
                  metrics_by_horizon: { "1Y": {...}, ... },   # EXPANDED to 10Y
                  equity: {dates:[], values:[]} } ],
  horizons_grid: { ...existing cells grid for the per-watchlist × bb-variant tables, +10Y... }
}
```

## Backend work

1. **`analysis/horizon_compare.py`**
   - Add `10y` to the horizon list everywhere (`strategy_horizons` + per-watchlist `cells`).
   - Expand `strategy_horizons` cells from `{xirr,sharpe,maxdd,cash}` to also include `final_value, mult, sortino, vol, maxdd_days` so Backtest cards/table are fully horizon-aware.

2. **`analysis/portfolio_view.py`**
   - New: `build_portfolio_nav()` - from `trades.csv` (Timed HODL rows) build daily cumulative shares per stock, multiply by historical daily close (six7 `_price_cache.pkl`, else `yf.download`), sum to a portfolio NAV series; also a cumulative-invested series. Derive `portfolio.horizon[H]` returns from the NAV series (trailing windows to today).
   - Replace HTML generation with **data emission**: write `docs/strat_data.js` (`window.STRAT_DATA = {...}`). Keep computing portfolio/backtest/iterations/horizons; reshape into the contract above.

3. **`analysis/build_web.py`** - confirm `data.js` carries 10Y; otherwise unchanged.

## Frontend work (/impeccable)

- Hand-author `docs/index.html` + CSS + JS as one unified app, designed by /impeccable (full visual latitude).
- Global horizon control with shared state; each section subscribes and re-renders.
- SVG (preferred) or themed-Chart.js chart kit covering: multi-series line (equity), area/line (drawdown, cash, NAV), grouped bars (regime), horizontal bars (buy dist, P&L), bars (deployment), donut (allocation), comparison tables with best-per-row highlight.
- Responsive + reduced-motion + contrast-checked per impeccable rules.
